# riffgrep BEXT Metadata Schema

## Executive Summary

riffgrep stores search fields in a custom packed format inside the BEXT Description field
(256 bytes at offset 0 of the BEXT chunk). This gives:

- **3-5 second search** across 1.2M files — only the first ~1KB per file is read
- Survives SoundMiner re-embedding (BEXT chunk stays in the file header)
- Links to SoundMiner database via **BEXT UMID field** (64 bytes, cryptographically unique)
- Keeps critical search fields in the file header (BEXT Reserved = waveform peaks)

---

## Complete BEXT Chunk Layout

**Standard BEXT Fields:**

```
Offset  Size  Field                 Usage
------  ----  -------------------   ----------------------------------------
0       256   Description           riffgrep packed schema (see below)
256     32    Originator            Vendor (e.g., "Samples From Mars")
288     32    OriginatorReference   Library (e.g., "DX100 From Mars")
320     10    OriginationDate       YYYY-MM-DD (preserve if present)
330     8     OriginationTime       HH:MM:SS (preserve if present)
338     8     TimeReference         Sample count (preserve if present)
346     2     Version               2 — EBU Tech 3285 (BWF); signals riffgrep schema
348     64    UMID                  SM _UMID (32 hex chars as ASCII) — migration receipt
412     10    Loudness Fields       Used for u8 waveform peaks (nonstandard; see below)
422     180   Reserved              180 u8 waveform peak values (RiffgrepU8 format)
```

**BEXT Version field (offset 346-347):** Set to **2** in riffgrep-packed files.
- Signals EBU Tech 3285 conformance, encouraging other software to treat Reserved as opaque.
- Honestly marks the nonstandard use of Loudness Fields (offsets 412-421) for u8 peaks.

**UMID field (offset 348-411):** Stores SoundMiner's `_UMID` hex string (32 ASCII chars).
This is the **sole SM cross-reference** and the ETL migration receipt. Non-empty = already ported.
`riffgrep` no longer stores SM's internal `recid`. Query: `SELECT * FROM justinmetadata WHERE _UMID = '{umid}'`

## Description Field — Packed Schema

Detected when: `version_major == 1 && version_minor >= 1 && bext_version >= 2 && file_id != 0`.
All four conditions must hold to avoid false positives from third-party files.

```
Byte Range    Size   Type         Field             Notes
----------    ----   -----------  ----------------  ----------------------------
[000:008]     8      uint64_be    file_id           High 64 bits of UUID v7 generated
                                                    at first pack. Never zero.
                                                    Bits 63-16: ms Unix timestamp
                                                    Bits 15-12: UUID version nibble (0x7)
                                                    Bits 11-0:  12-bit random suffix
                                                    Extract creation time: ts_ms = file_id >> 16
[008:010]     2      uint16_le    Version-Major     1 — signals binary UUID content
[010:012]     2      uint16_le    Version-Minor     2 — MARKERSv2 format
[012:044]     32     binary       MARKERSv2         Two 16-byte marker banks (see below)
[044:076]     32     ASCII        COMR/Comment      Free text (null-padded)
[076:080]     4      ASCII        POPM/Rating       e.g., "****"
[080:084]     4      ASCII        TBPM/BPM          e.g., "164 " (null-padded)
[084:088]     4      ASCII        TCOM/Subcategory  e.g., "DEMO"
[088:092]     4      ASCII        TCON/Category     e.g., "LOOP"
[092:096]     4      ASCII        TIT1/Genre ID     e.g., "ACID"
[096:100]     4      ASCII        TIT2/Sound ID     e.g., "DHC "
[100:104]     4      ASCII        TIT3/Usage ID     e.g., "XPM "
[104:112]     8      ASCII        TKEY/Key          e.g., "A#m\0\0\0\0\0"
[112:116]     4      ASCII        Take              e.g., "67  "
[116:120]     4      ASCII        Track             e.g., "1   "
[120:128]     8      ASCII        Item              e.g., "12345678"
[128:256]     128    ASCII        Reserved          (padding / future use)
```

**file_id semantics:**
- Zero → file not yet packed by riffgrep (treat as unpacked on read)
- Non-zero → high 64 bits of UUID v7 generated by `init_packed_and_write_markers()` at
  first write. Never set by Lua scripts; always riffgrep-generated.
- Collision probability: ~1/370K for a 10M file collection (safe).

## MARKERSv2 Format (Description[12:44])

32 bytes = two 16-byte banks (Bank A + Bank B).

Each bank:
```
[0:4]   m1 (u32 LE) — absolute sample offset; 0xFFFFFFFF = unused
[4:8]   m2 (u32 LE)
[8:12]  m3 (u32 LE)
[12:13] nibbles — packed reps: byte0 = (S1<<4)|S2, byte1 = (S3<<4)|S4
                  0=skip, 1-14=repeat count, 15=infinite
[14:16] padding (zero)
```

See `doc/MARKER_SERIALIZATION.md` for the full specification.

## Waveform Peaks (BEXT Reserved, offsets 422-601)

180 `u8` values, each the absolute amplitude peak for a column of audio samples.
Stored directly in the BEXT Reserved field (nonstandard use, clearly marked by `bext_version=2`).

Detection: `packed == true && peaks_format == RiffgrepU8`.
Third-party files with non-zero Reserved but not packed schema are treated as `BwfReserved`
(opaque, not interpreted as peaks).

---

## Database Linkage

The BEXT UMID field (offset 348-411, 64 bytes) stores the SoundMiner `_UMID` hex string
as ASCII. This is the sole SM cross-reference — it serves as both:
- A **migration receipt**: non-empty = file already ported by the ETL script
- A **lookup key**: query SM DB by `_UMID` to retrieve original metadata

```sql
SELECT * FROM justinmetadata WHERE _UMID = '976132720e774b668c36826386ae6505'
```

riffgrep does not store SM's `recid`. The UMID field (64 bytes) is cryptographically
large enough to be collision-resistant; recid was SM-specific and changed if the DB was rebuilt.

## File States

**Type A: Never touched by SM**
- Layout: `fmt` + `BEXT` + `LIST/INFO` + `ID3v2` + `data`
- BEXT in header → fast search ✅

**Type B: Edited in SM**
- Layout: `fmt` + `BEXT` + `filr` + `data` + `ID3` + `SMED` + `LIST/INFO` + `iXML`
- BEXT still in header → fast search still works ✅

**Type C: riffgrep-optimized**
- Layout: `fmt` + `BEXT` + `data` (minimal)
- Fast search ✅, clean structure ✅

## Key Design Decisions

**Why Description field?** Fixed location (offset 0 of BEXT data), binary-safe, 256 bytes.

**Why UUID v7?** Monotonically increasing IDs that embed "when riffgrep first processed this
file" without a separate metadata field. The version nibble (0x7) makes bytes self-describing.

**Why ASCII for most fields?** Human-readable in a hex editor, consistent, negligible size
difference vs. binary for short text fields.

**Why bext_version=2?** Signals EBU Tech 3285 conformance (BWF), encourages respectful
treatment of Reserved field by other software, and honestly marks nonstandard use of
Loudness Fields (412-421) for u8 peak data.

**Status: Production-ready — schema locked before 1.2M file port.**
