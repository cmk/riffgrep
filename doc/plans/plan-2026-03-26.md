# AudioSource Trait Refactor Plan

## Goal

Replace scattered `if is_aiff(path)` branches with a trait-based dispatch.
Adding a new format (MP3, FLAC, OGG) should require zero changes outside
the new source implementation and a one-line registration.

---

## The Trait

```rust
pub trait AudioSource: Send + Sync {
    /// File extensions this source handles (lowercase, no dot).
    fn extensions(&self) -> &[&str];

    /// Read metadata from file headers.
    /// Fast path (<4KB read) where possible; full decode as fallback.
    fn read_metadata(&self, path: &Path) -> Result<UnifiedMetadata, RiffError>;

    /// Compute stereo RMS peaks: 180 left + 180 right = 360 u8 values.
    fn compute_peaks_stereo(&self, path: &Path) -> Result<Vec<u8>, RiffError>;

    /// Load full mono PCM (left channel, i16) for zoom rendering.
    fn load_pcm(&self, path: &Path) -> Result<PcmData, RiffError>;

    /// Audio format info (duration, sample rate, bit depth, channels).
    fn audio_info(&self, path: &Path) -> Result<AudioInfo, RiffError>;

    /// Write metadata changes back to the file.
    /// Returns None if the format is read-only.
    fn write_metadata(
        &self,
        _path: &Path,
        _before: &UnifiedMetadata,
        _after: &UnifiedMetadata,
        _force: bool,
    ) -> Option<anyhow::Result<()>> {
        None
    }
}
```

---

## Implementations

### `RiffSource` (WAV)

Extensions: `["wav"]`

- `read_metadata`: `scan_chunks` → `parse_bext` + `parse_riff_info` → `merge_metadata`
- `compute_peaks_stereo`: stream from `data` chunk via `FmtChunk` (fast, no full decode)
- `load_pcm`: direct read from `data` chunk offset
- `audio_info`: parse `fmt` chunk + `data` size
- `write_metadata`: surgical BEXT offset writes (existing `write_metadata_changes`)

### `DecoderSource` (AIFF, MP3, FLAC, OGG, etc.)

Extensions: `["aif", "aiff", "mp3", "flac", "ogg"]`

- `read_metadata`: returns minimal `UnifiedMetadata` (path only; metadata via DB enrichment or ID3)
- `compute_peaks_stereo`: full decode via `rodio::Decoder`, RMS binning
- `load_pcm`: full decode, extract left channel
- `audio_info`: full decode, count frames
- `write_metadata`: returns `None` (read-only)

### Future: `FlacSource`, `Mp3Source` (optional specializations)

Only needed if we want to read native metadata (Vorbis comments, ID3v2)
without full decode. Can extend `DecoderSource` or implement `AudioSource`
directly. Not required for initial refactor.

Walker globs derived from `registry.all_extensions()` — eliminates the

## Prop-testable Properties

All properties live in `src/engine/source.rs` under `mod proptests`.
Follow the existing codebase conventions:

- Domain-aware generators with weighted `prop_oneof!` for edge cases
- `proptest!` macro blocks with `ProptestConfig::with_cases(512)`
- Properties prefixed `proptest_` with `///` doc comments stating the invariant
- Grouped in `mod proptests` inside `#[cfg(test)] mod tests`

### Generators

    proptest! {
        #![proptest_config(ProptestConfig::with_cases(512))]

        /// P1: Peaks are exactly 360 bytes or empty.
        #[test]
        fn proptest_peaks_length(path in arb_test_file()) {
            let registry = AudioRegistry::new();

        fn proptest_peaks_normalization(path in arb_test_file()) {
            let registry = AudioRegistry::new();
            if let Some(source) = registry.for_path(&path) {
                if let Ok(peaks) = source.compute_peaks_stereo(&path) {
                    if peaks.len() == 360 {

            let registry = AudioRegistry::new();
            if let Some(source) = registry.for_path(&path) {
                if source.audio_info(&path).is_ok() {
                    let pcm = source.load_pcm(&path);
                    prop_assert!(pcm.is_ok(), "audio_info ok but load_pcm failed: {:?}", pcm.err());

                }
            }

        /// P4: Audio info fields are internally consistent.
        #[test]
        fn proptest_audio_info_consistency(path in arb_test_file()) {
            let registry = AudioRegistry::new();
            if let Some(source) = registry.for_path(&path) {
                if let Ok(info) = source.audio_info(&path) {
                    prop_assert!(info.sample_rate > 0);
                    prop_assert!(info.channels > 0);
                    prop_assert!(info.total_samples > 0);
                    let expected_dur = info.total_samples as f64 / info.sample_rate as f64;
                    prop_assert!(
                        (info.duration_secs - expected_dur).abs() < 0.01,
                        "duration_secs={} but total/rate={}", info.duration_secs, expected_dur
                    );
                }
            }
        }

        /// P5: Metadata always preserves the input path.
        #[test]
        fn proptest_metadata_path_preserved(path in arb_test_file()) {
            let registry = AudioRegistry::new();
            if let Some(source) = registry.for_path(&path) {
                if let Ok(meta) = source.read_metadata(&path) {
                    prop_assert_eq!(meta.path, path);
                }
            }
        }

        /// P7: Peak computation is deterministic.
        #[test]
        fn proptest_peaks_deterministic(path in arb_test_file()) {
            let registry = AudioRegistry::new();
            if let Some(source) = registry.for_path(&path) {
                let a = source.compute_peaks_stereo(&path);
                let b = source.compute_peaks_stereo(&path);
                match (a, b) {
                    (Ok(pa), Ok(pb)) => prop_assert_eq!(pa, pb),
                    (Err(_), Err(_)) => {} // both failed — ok
                    _ => prop_assert!(false, "one succeeded, one failed"),
                }
            }

### P10: Fast Path Equivalence (the key property)

```rust
    /// P10: RiffSource and DecoderSource produce near-identical peaks for WAV files.
    ///
    /// Tolerance: ±2 per bin due to rounding differences between direct
    /// i16/i24 PCM reads (RiffSource) and rodio's f32 decode path (DecoderSource).
    #[test]
    fn proptest_fast_path_equivalence() {
        let riff = RiffSource;
        let decoder = DecoderSource;
        let wav_files = [
            "test_files/clean_base.wav",
            "test_files/all_riff_info_tags_with_numbers.wav",
            "test_files/all_riff_info_tags_with_numbers-sm.wav",
        ];
        for path_str in &wav_files {
            let path = Path::new(path_str);
            if !path.exists() { continue; }

            let riff_peaks = riff.compute_peaks_stereo(path).unwrap();
            let decoder_peaks = decoder.compute_peaks_stereo(path).unwrap();

            assert_eq!(riff_peaks.len(), decoder_peaks.len(),
                "length mismatch for {path_str}");

            for (i, (a, b)) in riff_peaks.iter().zip(decoder_peaks.iter()).enumerate() {
                let diff = (*a as i16 - *b as i16).abs();
                assert!(diff <= 2,
                    "{path_str} bin {i}: riff={a} decoder={b} diff={diff} (max 2)");
            }
        }
    }
```

This test is not proptest-driven (finite file set) but validates the
critical invariant: the RIFF fast path is a correct optimization of the
decoder path. If this fails, either the RIFF parser or the decoder
normalization has a bug.

---

## Non-goals

- **Format-specific metadata parsing** (Vorbis comments, ID3v2 on MP3):
  deferred. Metadata comes from DB enrichment for now.
- **Write support for non-WAV formats**: not planned. BEXT is WAV-specific.
- **Streaming peak computation**: decoder path currently loads entire file.
  Acceptable for reference tracks; revisit if SUB-scale collections have
  non-WAV files.

---

## Risks

- `rodio::Decoder` may not support all symphonia formats depending on
  feature flags. Currently enabled: `wav`, `pcm`, `aiff`. Need to add
  `mp3`, `flac`, `ogg` features to Cargo.toml.
- The decoder path is ~10x slower than the RIFF path for peaks/zoom
  because it decodes the entire file. Acceptable for reference collections
  but would be a bottleneck for indexing 1.2M files.
- `PcmData` loads the entire file into memory for zoom. For long tracks
  (>10min) this could be 100MB+. Consider streaming or downsampling.
