# Plan 11 — Serialize/Deserialize Retrofit on Markers + Playback FSMs

## Goal

Derive `Serialize + Deserialize` on `MarkerFsmState`,
`PlaybackFsmState`, and every `Input` / `Output` variant in those
two modules, so stdio-core's form-patch protocol can snapshot and
restore all three sub-FSMs (markers, playback, search) on equal
footing.

## Context

Per `stdio-core/doc/designs/tui-fsm.md` §1 (load-bearing
consideration #1: "keep state serializable"), the FSM wrapper must
round-trip through serde for stdio-core's form protocol to land
naturally. Plan 08's `search_fsm` was built this way from day one;
markers (Plans 03–04) and playback (Plan 06) predate the
stdio-core design doc and don't have serde derives.

This is a mechanical pass. The current state types are all
trivially serializable: `Copy` scalars (`bool`, enums) plus
`MarkerConfig` (which is `Copy` and contains only u32 / u8 / `[u8;
4]`). No opaque handles, no references, no channels. Path types
in `TypedAction::LoadSample(PathBuf)` are already serde-ready
(the search module demonstrates).

## Tasks

### 1. Audit `MarkerConfig` serde readiness

`src/engine/bext.rs` defines `MarkerConfig` + `MarkerBank`. Add
`Serialize + Deserialize` derives. Both are `Copy` structs of
primitives — derive is one line each.

### 2. Derive serde on `MarkerFsmState` + `marker_fsm::Input` +
   `marker_fsm::Output`

`src/engine/marker_fsm.rs`:
- `MarkerFsmState` (struct of scalars + `MarkerConfig`).
- `Selection` (enum, unit variants).
- `Bank` (enum, unit variants — already exists).
- `Input` (14 variants; most unit, one `PathBuf` carrier for
  `ExportMarkersCsv` / `ImportMarkersCsv`, one `MarkerConfig`
  carrier for `LoadConfig`). `PathBuf` is serde-ready.
- `Output` (`WriteCsv(PathBuf)` / `ReadCsv(PathBuf)`).

### 3. Derive serde on `PlaybackFsmState` + `playback_fsm::Input` +
   `playback_fsm::MixerCommand`

`src/engine/playback_fsm.rs`:
- `PlaybackFsmState` (struct of scalars).
- `Transport` (enum, unit variants).
- `Input` (14 variants, mix of unit + `u32` + `bool` carriers —
  all primitive).
- `MixerCommand` (4 unit variants).

### 4. Property extensions for round-trip

Mirror Plan 08's R6:
- P_serde for markers: state round-trips through `serde_json`
  for any reachable `MarkerFsmState`.
- Q_serde for playback: same for `PlaybackFsmState`.

Add one `proptest!` test per FSM module under its existing
`tests/engine/<name>/prop.rs`.

### 5. Tiny sanity test

For each of the three FSMs (markers, playback, search), a unit
test that serializes every `Input` variant and round-trips it.
Search already has this
(`input_and_output_serde_round_trip`); add the equivalent for
markers + playback.

## Verification

### Properties

Existing P1–P8 (markers) and Q1–Q8 (playback) unchanged. New
serde round-trip properties pass for both.

### Spot checks

- Unit test for every `Input` variant of both FSMs round-trips
  through `serde_json::to_string` → `from_str`.

### Build gates

- `cargo build` — no errors
- `cargo test` — all pass
- `cargo clippy --all-targets -- -D warnings` — clean
- `cargo fmt --check` — clean

## Deferred

None. This closes the stdio-core-embedding prerequisite surface; all
three sub-FSMs become form-patch compatible.
