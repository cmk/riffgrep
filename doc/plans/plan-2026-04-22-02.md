# Plan 07 — Playback Reverse Path Unification

## Goal

Route `PlaybackEngine` mutators through `PlaybackFsm` (finishing Task 3
deferred from Plan 06), and collapse the two reverse-playback code paths
— the **pre-reverse buffer copy** and the **runtime atomic flag** — into
one path via `effective = segment.reversed ^ global_reversed`.

## Context

Plan 06 (`plan-2026-04-18-04.md`) shipped the FSM module + property suite
but explicitly deferred engine wiring to this sprint so the same
call-sites wouldn't be touched twice. The FSM currently carries
`#[allow(dead_code)]` on `Input`, `MixerCommand`, and `PlaybackFsm`
(`src/engine/playback_fsm.rs:73, 127, 269`) waiting for this sprint to
consume them.

The reverse-playback debt is documented in `doc/designs/debt-playback.md`:

- **Path 1 — pre-reverse buffer copy** (`src/engine/playback.rs:528–534`):
  `play_with_segments` materializes a reversed frame region into a scratch
  buffer, then plays it forward. Requires a `logical_start` remap so the UI
  can report sensible positions.
- **Path 2 — runtime atomic flag** (`src/engine/playback.rs:198, 331`):
  `on_frame_boundary` reads `control.reversed` to swap origin/boundary,
  and `next()` steps `pos` backward by `2 * channels`. Works regardless
  of path 1.

The two paths can compose incorrectly (the debt doc calls out `pending_restart`
using `first.start` / `first.logical_start` where reverse mode needs
`first.end - 1`). Unifying on path 2 — with a per-segment `reversed: bool` and
the XOR identity — removes the scratch-region dance, the `logical_start`
remap, and the composition bug in one pass. The FSM's `reversed` field
(already in `PlaybackFsmState`) becomes the single source of truth for the
global flag; `PlaySegment.reversed` is the per-segment override.

## Tasks

### T1 — Add `reversed: bool` to `PlaySegment`

**Problem:** `PlaySegment` (`src/engine/playback.rs:96`) has no per-segment
reverse flag. The current representation relies on `logical_start` to remap
a pre-reversed scratch region back to source-file positions.

**Solution:** Add `pub reversed: bool` (default `false`) to `PlaySegment`.
Drop `logical_start` — with path-1 gone, frame reporting uses the raw
buffer position and the UI maps per-segment direction.

**Types:**
```rust
pub struct PlaySegment {
    pub start: u32,
    pub end: u32,
    pub reps: u32,
    pub reversed: bool,        // new
    // logical_start: u32,     // removed
}
```

### T2 — Remove the pre-reverse buffer copy

**Problem:** `play_with_segments` at `src/engine/playback.rs:528–534`
materializes a reversed frame region into a scratch buffer:
```rust
let reversed: Vec<f32> = samples[lo..hi]
    .chunks(ch)
    .rev()
    .flat_map(|frame| frame.iter().copied())
    .collect();
```
This doubles memory for reversed segments, forces the `logical_start` remap,
and is redundant with path 2.

**Solution:** Delete the scratch-region path. The segment's audio data is
the forward buffer; `reversed` drives traversal direction at runtime
(T3).

### T3 — Compute effective direction on segment entry

**Problem:** With path 1 gone, `on_frame_boundary`
(`src/engine/playback.rs:198`) needs to decide traversal direction per
segment, not once globally.

**Solution:** At segment entry, compute
`effective = segment.reversed ^ global_reversed` (where `global_reversed`
comes from `PlaybackFsmState.reversed`) and store it on
`SourceControl::reversed`. `next()` (`src/engine/playback.rs:331`) is
unchanged — it already reads the atomic and steps backward when set.
The single atomic now reflects the XOR'd direction rather than the
bare global flag.

**API surface:** no public change. Internal: `SegmentSource` gains a
reference (or callback) to pull `global_reversed` from the FSM when
crossing a segment boundary.

### T4 — Wire `PlaybackEngine` through `PlaybackFsm`

**Problem:** `PlaybackEngine::{set_reversed, set_loop_enabled, seek_to_sample,
restart_program, toggle_pause, stop, play_with_segments}` mutate atomics
directly. The FSM is inert.

**Solution:** Add `PlaybackEngine::fsm: Mutex<PlaybackFsm>` (TUI-facing API
is single-threaded; `Mutex` is for interior mutability only — no contention
expected). Each mutator becomes `consume` → apply atomics from the new
state, keeping public signatures unchanged:

| Method | FSM input |
|---|---|
| `set_reversed(v)` | `Input::SetReverse(v)` |
| `set_loop_enabled(v)` | `Input::SetLoop(v)` |
| `seek_to_sample(p)` | `Input::Seek(p)` |
| `restart_program()` | `Input::Restart` (no-op from Stopped per Q7) |
| `toggle_pause()` | `Input::Pause` / `Input::Resume` dispatched per current transport |
| `stop()` | `Input::Stop` |
| `play_with_segments(…)` | `Input::Play` + segment-list install |

Mixer-thread consumption points:
- Segment boundary crossed → `Input::SegmentEnded`
- Last segment ended → `Input::ProgramEnded`
- Seek applied → `Input::ConsumeSeek`
- Restart applied → `Input::ConsumeRestart`

Drop `#[allow(dead_code)]` from `Input`, `MixerCommand`, and `PlaybackFsm`
(`src/engine/playback_fsm.rs:73, 127, 269`).

### T5 — UI call-site audit

**Problem:** Confirm nothing bypasses the engine to touch atomics directly.

**Solution:** Survey `src/ui/mod.rs`. Expected call-site counts (from pre-sprint
trace): `set_loop_enabled` ×3, `seek_to_sample` ×9, `play_with_segments` ×3,
`stop` ×8, `set_reversed` ×1, plus pauses/resumes. Public API unchanged, so
no functional diff expected — verify there are no stray `engine.control.*`
or `SourceControl` field writes outside `playback.rs`.

### T6 — Reverse-mode `pending_restart` fix

**Problem:** `pending_restart` resolution currently uses
`first.start` / `first.logical_start` (per `debt-playback.md:83–86, 114–115`).
After T1–T3, `logical_start` is gone, and the correct origin in reverse
mode is `first.end - 1`.

**Solution:** Route restart through the FSM's `ConsumeRestart`. When the
mixer picks it up, it resolves the origin based on the first segment's
effective direction: `origin = if effective { first.end - 1 } else { first.start }`.

### T7 — Reverse-path properties in `tests/engine/playback_fsm/`

**Problem:** Plan 06's Q1–Q8 cover transport and intent plumbing but not
reverse-direction semantics.

**Solution:** Extend `generators.rs` with a `PlaySegment` strategy that
includes `reversed` at `prop_oneof![Just(false) => 3, Just(true) => 1]`
(boundary-bias toward forward, but reverse is reachable), and add
properties R1–R6 in `prop.rs` (table below). Migrate
`test_segment_source_reverse_segment` (`src/engine/playback.rs:1514–1518`)
to `unit.rs` if it still covers anything path-2 doesn't after T2.
