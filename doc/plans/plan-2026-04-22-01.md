# Cargo-docs hosting (GitHub Pages)

## Goal

Publish rustdoc for `riffgrep` to GitHub Pages on every push to `main`, served
at `https://cmk.github.io/riffgrep/riffgrep/`.

## Context

This is a task sprint, not an FSM-roadmap sprint — no Plan NN, no property
suite. It prefaces Plan 07 (`plan-2026-04-22-02.md`) so that by the time Plan
07 lands the engine-wiring changes, the published docs already reflect the
new `PlaybackEngine` / `PlaybackFsm` surface and reviewers can browse the API
in a rendered form.

Today `cargo doc` is only usable locally. Two consequences: (1) reviewers on
PRs have to check out the branch to read doc-comments in context; (2) the
crate's growing public surface (FSMs, engine, similarity) has no rendered
reference for the README or future README screenshots.

CI (`.github/workflows/ci.yml`) covers build/clippy/test/fmt on both push and
PR. Docs deployment is a **push-to-main-only** job: GitHub Pages has a
single live deployment target, and per-PR previews add an `environment` +
cleanup surface we don't need yet.

## Tasks

### T1 — Add `.github/workflows/docs.yml`

**Problem:** No workflow builds or publishes rustdoc today.

**Solution:** New workflow with two jobs.

- Trigger: `push` to `main` only. `workflow_dispatch` for manual retries.
- Top-level `permissions: contents: read`. Job-level `pages: write` +
  `id-token: write` on the deploy job.
- `concurrency: group: pages, cancel-in-progress: false` — GitHub's
  recommended default so a slow docs build isn't cancelled mid-upload by a
  newer push.

**Job `build`:**
- `actions/checkout@v4` (no LFS needed for doc generation).
- `sudo apt-get install -y libasound2-dev` — matches `ci.yml:25`; rodio's
  transitive alsa dep is required at link time even for doc generation.
- `dtolnay/rust-toolchain@v1` pinned to `1.94.0` (matches `ci.yml:29`).
- `Swatinem/rust-cache@v2` — shares cache layout with the existing CI job.
- `cargo doc --no-deps --workspace` — `--no-deps` keeps the artifact small
  and focused on riffgrep's own surface; workspace flag is a no-op today
  (single crate) but future-proofs.
- Inject a top-level redirect at `target/doc/index.html`:
  ```html
  <meta http-equiv="refresh" content="0; url=riffgrep/index.html">
  ```
  so the Pages root URL lands on the crate's index instead of the bare
  directory listing that GitHub now blocks.
- `actions/upload-pages-artifact@v3` with `path: target/doc`.

**Job `deploy`:**
- `needs: build`
- `environment: { name: github-pages, url: ${{ steps.deployment.outputs.page_url }} }`
- `actions/deploy-pages@v4` (step id `deployment`).

### T2 — Pages enablement (manual, one-time)

**Problem:** `actions/deploy-pages@v4` fails if Pages isn't configured to use
GitHub Actions as its source.

**Solution:** Document in the PR body that, before the first merge, the repo
owner must visit **Settings → Pages → Build and deployment → Source** and
select **GitHub Actions**. The workflow fails loudly with a clear error if
this step is skipped, so no silent breakage. No code change needed for this
task; it's a checklist item for the PR.

## Verification

This sprint has no property-testable surface. Verification is end-to-end.

### Spot checks

- Local: `cargo doc --no-deps` completes with exit 0. Note any rustdoc
  warnings (left as-is for this sprint; `RUSTDOCFLAGS=-D warnings` is
  deferred).
- PR preview: pushing the branch triggers `ci.yml` as normal and does **not**
  trigger `docs.yml` (push-to-main-only). Confirm in the Actions tab.
- Post-merge: `docs.yml` runs green; the Actions summary links to the live
  URL; `https://cmk.github.io/riffgrep/` 302s to `…/riffgrep/index.html` and
  the crate index renders; `PlaybackFsm`, `PlaybackEngine`, `MarkerFsm`, and
  `search_fsm` are navigable.

### Build gates

- `cargo build` — unchanged; not exercised by this sprint.
- `cargo doc --no-deps` — exit 0 locally before pushing.
- `cargo test` / `cargo clippy` / `cargo fmt --check` — unchanged by this
  sprint but pre-commit hook still runs them on the plan commit and the
  workflow commit.

## Deferred

- **PR-preview docs** (per-PR deploys to subpaths like
  `https://cmk.github.io/riffgrep/pr/123/`). Needs a second environment,
  subpath routing, and tag-based cleanup on PR close. Skip until a reviewer
  actually asks.
- **`--document-private-items`.** Default `--no-deps` keeps the artifact
  scoped to the public surface. Flip when we have a documented reason.
- **`RUSTDOCFLAGS: -D warnings`.** Defer until we've cleaned up any current
  doc warnings; adding this blind risks turning the first post-merge run red.
- **Docs-status badge in README.** Easy win but not part of the minimum
  deliverable; can ride with the first README refresh sprint.
