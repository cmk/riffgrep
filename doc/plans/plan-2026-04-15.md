# Embedding & Integration Roadmap

## Context

riffgrep has a working brute-force similarity search prototype (b01db24)
with known issues from the Copilot and local reviews. The goal is to
harden it, then integrate riffgrep into stdio as an MCP-accessible tool
alongside the existing device drivers (driver-mpc, driver-qu, etc.).

The agentic design doc from Gemini identifies the right end-state: an
LLM agent queries riffgrep's search/similarity/metadata tools through
MCP, with PQ-accelerated search keeping latency under 30ms. But the
path there is 4 discrete sprints.

stdio's architecture: per-driver crates with `#[tool_router]` /
`#[tool]` macros from rmcp. No shared device trait. Drivers are merged
via `ToolRouter::merge()` in the server crate. Submodules in `ext/` are
excluded from the workspace and patched via `[patch.crates-io]`.

iffgrep is a separate thing (SysEx/IFF CLI using driver-sysex).
riffgrep will be a git submodule in stdio's `ext/`, with a thin
`crates/driver-rfg` wrapper exposing MCP tools. Eventually a shared
core crate will be extracted for both riffgrep and iffgrep.

## Plan 1: Harden embedding prototype (riffgrep repo)

**Branch:** `feat/embeddings` (already exists)

Address the 6 findings from the b01db24 review:

1. **`insert_embedding` silent no-op** (`sqlite.rs:633`) — check
   rows-affected, return error when path not found.

2. **`load_all_embeddings` RAM bomb** (`ui/mod.rs:1528`) — add
   row-count guard before loading. If corpus > 100K, show status
   message and abort. Long-term fix is PQ (Plan 3).

3. **`load_all_embeddings` uses `prepare`** (`sqlite.rs:670`) —
   switch to `prepare_cached` for consistency.

4. **`sim` score doc is misleading** (`similarity.rs:54`) — update
   doc comment to say "window-relative [0.0, 1.0], not absolute.
   Use `dist` for cross-query comparisons."

5. **Schema version not bumped** (`sqlite.rs:822`) — bump
   `SCHEMA_VERSION` to 2, add migration branch:
   `ALTER TABLE samples ADD COLUMN embedding BLOB`.

6. **Stale docstring on squared distance** (`similarity.rs:29`) —
   fix comment to match actual `l2_distance` usage.

**Files:** `src/engine/sqlite.rs`, `src/engine/similarity.rs`,
`src/ui/mod.rs`

**Verification:** `cargo test --lib`, existing similarity proptests
still pass, new test for insert_embedding error on missing path.

## Plan 2: Extract headless engine API (riffgrep repo)

**Branch:** `feat/engine-api`

The TUI and CLI currently call engine functions directly. For MCP
integration, we need a clean programmatic API that doesn't depend on
crossterm/ratatui.

1. **Create `src/engine/api.rs`** — public functions that the MCP
   driver will call:
   - `search(roots, query, limit) → Vec<UnifiedMetadata>`
   - `similar(db_path, path, limit) → Vec<SimilarityResult>`
   - `metadata(path) → UnifiedMetadata`
   - `index(roots, db_path) → IndexStats`

2. **Ensure `riffgrep` crate is a lib+bin** — `src/lib.rs` re-exports
   `engine::api`. The binary (`src/main.rs`) uses the lib. Already
   structured this way.

3. **Add `--json` output for `--similar`** — machine-readable output
   for agent consumption. Already partially there (cli::OutputMode::Json).

**Files:** `src/engine/api.rs` (new), `src/engine/mod.rs`, `src/lib.rs`

**Verification:** `cargo test`, new integration test calling API
functions directly without TUI.

## Plan 3: PQ acceleration (riffgrep repo)

**Branch:** `feat/pq`

Replace brute-force scan with Product Quantization for sub-30ms search
at 1.2M scale. This is the core of what the agentic doc describes.

1. **PQ training** — k-means on stored embeddings to build codebook.
   128 sub-spaces × 256 centroids. Store codebook in `metadata` table.

2. **PQ encoding** — compress each 512-dim f32 vector to 128-byte
   code. Store codes in new `pq_code BLOB` column.

3. **ADC search** — build LUT from query vector + codebook, scan
   codes with table lookups. Use rayon for parallelism. No SIMD
   intrinsics in v1 — profile first, optimize later.

4. **Fallback** — if codebook absent (fresh DB), fall back to
   brute-force with the existing path.

**Files:** `src/engine/pq.rs` (new), `src/engine/similarity.rs`,
`src/engine/sqlite.rs`

**Verification:** proptest that PQ ranking preserves top-10 from
brute-force (within tolerance). Benchmark comparing brute-force vs
PQ at 10K/100K/1M synthetic embeddings.

## Plan 4: stdio integration (stdio repo)

**Branch:** `feat/driver-rfg` in stdio

Wire riffgrep into stdio as an MCP driver via git submodule.

1. **Add submodule** — `git submodule add <riffgrep-url> ext/riffgrep`.
   Exclude from workspace. Add `[patch.crates-io]` entry pointing to
   `ext/riffgrep`.

2. **Create `crates/driver-rfg`** — thin wrapper:
   ```rust
   #[tool_router]
   impl RfgDriver {
       #[tool(description = "Search audio samples by metadata")]
       async fn rfg_search(&self, query: String, ...) -> CallToolResult

       #[tool(description = "Find similar sounds by embedding")]  
       async fn rfg_similar(&self, path: String, limit: u32) -> CallToolResult

       #[tool(description = "Read metadata for an audio file")]
       async fn rfg_metadata(&self, path: String) -> CallToolResult
   }
   ```

3. **Add feature flag** — `rfg = ["dep:driver-rfg"]` in server's
   Cargo.toml. Merge into `ToolRouter` in server main.

4. **Wire up** — `driver-rfg` depends on `riffgrep` (the lib crate)
   and calls `engine::api::*` functions.

**Files (stdio repo):**
- `.gitmodules` — add riffgrep submodule
- `Cargo.toml` — workspace exclude + patch
- `crates/driver-rfg/` — new crate
- `crates/server/Cargo.toml` — feature flag
- `crates/server/src/main.rs` — merge router

**Verification:** `cargo build --features rfg`, manual test with
Claude Code MCP client calling rfg_search.

---

## Execution order

Plans 1 → 2 are independent of stdio (riffgrep repo only).
Plan 3 depends on Plan 1 (needs schema migration).
Plan 4 depends on Plan 2 (needs headless API).

```
Plan 1 (harden) ──→ Plan 3 (PQ)
Plan 2 (API)    ──→ Plan 4 (stdio integration)
```

Plans 1 and 2 can run in parallel.
