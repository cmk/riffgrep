# Plan 07 — Playback FSM, Reverse-Path Unification + Engine Wiring

## Goal

Two fused concerns: (a) wire the `playback_fsm::PlaybackFsm` +
`MixerCommand` output into `PlaybackEngine` so UI-facing
transitions dispatch through the FSM, and (b) collapse the two
independent reverse-playback code paths per
`doc/designs/debt-playback.md` so the FSM's `reversed` field is
the single source of truth for playback direction.

## Context

Plan 06 (`plan-2026-04-18-04.md`, PR #20) shipped the pure FSM +
property suite. Its Status block explicitly deferred Task 3:
"PlaybackEngine wiring needs the same call-sites that Plan 07
rewrites for reverse-path unification; bundling them avoids a
double-touch." That's this sprint.

The reverse-path story from `doc/designs/debt-playback.md`:

1. **Path 1 — Pre-reverse buffer copy** (`play_with_segments`):
   marker-ordered reversed segments copy frames into a scratch
   region in reverse, then play forward through the scratch.
2. **Path 2 — Atomic reversed flag** (`SourceControl::reversed` +
   `SegmentSource::next()` + `on_frame_boundary()`): runtime
   reverse toggle steps the iterator backwards and swaps
   origin/boundary per segment.

Both paths can be active simultaneously. A marker-ordered
reversed segment with the runtime flag on would double-reverse.
The design doc's "Unification Plan" collapses both into the
atomic-flag approach using
`effective = segment.reversed ^ global_reversed`.

## Tasks

### 1. Wire `PlaybackFsm` into `PlaybackEngine`

Add `fsm: Mutex<PlaybackFsm>` (or equivalent single-threaded owner)
to `PlaybackEngine`. Each UI-facing method dispatches through it:

- `play_with_segments` → `Input::Play` + handle
  `MixerCommand::Start`
- `pause` → `Input::Pause` + `MixerCommand::Pause`
- `resume` → `Input::Resume` + `MixerCommand::Resume`
- `stop` → `Input::Stop` + `MixerCommand::Stop`
- `seek_to(pos)` → `Input::Seek(pos)`; mixer consumes via
  `ConsumeSeek` on next frame boundary
- `restart_program` → `Input::Restart`; mixer consumes via
  `ConsumeRestart`
- `set_reversed(v)` → `Input::SetReverse(v)`
- `set_loop_enabled(v)` → `Input::SetLoop(v)`

Atomic state (`pending_seek`, `pending_restart`, `reversed`,
`loop_enabled` on `SourceControl`) is projected from the FSM
state after each dispatch. The mixer thread continues to read
atomics; the FSM is the single-threaded coordinator.

### 2. Mixer signals back to the FSM

`SegmentSource::on_frame_boundary` fires `Input::SegmentEnded` /
`ProgramEnded` when segments or the playlist complete. The
`pending_seek.swap` / `pending_restart.swap` paths fire
`Input::ConsumeSeek` / `ConsumeRestart` when the atomics clear.
These inputs route through a cross-thread channel (mpsc or
equivalent) back to the FSM owner.

### 3. Collapse the dual reverse paths

Per `debt-playback.md` §Unification Plan:

- Remove the pre-reverse buffer copy from `play_with_segments`.
- Store a per-segment `reversed: bool` on `PlaySegment`.
- When entering a segment, compute
  `effective = segment.reversed ^ global_reversed` and store on
  `SourceControl::reversed`.
- Remove `logical_start` mapping — frame reporting uses the real
  buffer position; UI maps by segment direction.

### 4. Fix reverse-mode seek + restart

Per `debt-playback.md` §Risks:

- `pending_seek` targets file-space frames. After buffer-copy
  removal, seek targets map directly to buffer positions.
- `pending_restart` must use `first.end - 1` (not `first.start`)
  when `effective` is reversed.

### 5. Populate `tests/engine/playback_fsm/unit.rs`

Per Plan 06 §7 cleanup: this file was a placeholder. Now fill it
with the `SegmentSource` regression tests for reverse-mode
behavior. From `debt-playback.md` §Testing:

- Forward segment + global reverse = reversed playback.
- Reversed segment + global reverse = forward playback (XOR).
- Crossfade at loop boundary in reverse mode.
- Seek during reversed playback lands at correct position.
- Restart during reversed playback starts from correct end.

### 6. Property extensions for reverse invariants

Add to `tests/engine/playback_fsm/prop.rs`:

- Q9: `effective = segment.reversed ^ global_reversed` is
  idempotent under double-toggle of `SetReverse`.
- Q10: After `Restart` + `ConsumeRestart` in reversed mode, the
  mixer's next frame emits from `playlist[0].end - 1`.

### 7. Cleanup

- Remove the narrow `#[allow(dead_code)]` attributes on
  `playback_fsm::Input`, `MixerCommand`, `PlaybackFsm`.
- `debt-playback.md` gets a "Resolved (YYYY-MM-DD)" status block
  at the top; the Unification Plan section becomes historical.
