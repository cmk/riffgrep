# Plan 10 — Input-Mode FSM (Normal ↔ Insert)

## Goal

Formalize the TUI input mode (`Normal` ↔ `Insert`) and the key
routing it governs as a small rust-fsm state machine, matching the
markers / playback / search sub-FSM pattern. Lifts keybindings out
of the mode flag's implicit control so stdio-core can route events
without replicating the mode check.

## Context

Today `App::input_mode: InputMode { Normal, Insert }` gates key
routing in `src/ui/mod.rs` (every keystroke branches on the flag
to decide whether to route to the search bar or the action
dispatcher). Per `stdio-core/doc/designs/tui-fsm.md` §4
("keybindings above the FSM"), this gate is a separate concern
from any single sub-FSM — it decides *which* FSM sees an event.

The sub-region is small but load-bearing: it's the top-level
router every other sub-FSM depends on.

## Tasks

### 1. FSM module `src/engine/input_mode_fsm.rs`

State: `InputModeState { mode: Mode, last_mode: Mode }` where
`Mode = Normal | Insert`. `last_mode` lets `Esc` always return to
Normal while still supporting `EnterInsert`-from-Insert idempotence.

Input (small enum):
- `EnterInsert` — `Normal → Insert`.
- `EnterNormal` — `Insert → Normal`.
- `Toggle` — flip.
- `Reset` — unconditionally back to Normal (Esc / startup).

Output: none — this FSM is pure state. The App reads `mode()` and
routes key events accordingly.

Derive `Serialize + Deserialize` for stdio-core embedding (form-patch
may want to snapshot the mode).

### 2. Property suite

- I1: `EnterInsert ∘ EnterInsert ≡ EnterInsert` (idempotence).
- I2: `EnterNormal ∘ EnterInsert ∘ EnterNormal ≡ Normal`.
- I3: `Reset` always lands Normal.
- I4: `Toggle ∘ Toggle ≡ id` (self-inverse).
- I5: Serde round-trip.

### 3. App integration

Replace `App.input_mode: InputMode` field with
`input_mode_fsm: InputModeFsm`. Migrate ~15 call-sites in
`src/ui/mod.rs` (accessor reads only — no data dependencies).

Key routing becomes:
```rust
match self.input_mode_fsm.mode() {
    Mode::Insert => self.search_runner.dispatch(
        Input::QueryChanged(c.to_string())
    ),
    Mode::Normal => if let Some(action) = self.keymap.resolve(key) {
        self.dispatch(action)
    }
}
```

### 4. Cleanup

- Remove `pub enum InputMode` from `src/ui/mod.rs`.
- Update `widgets.rs` to read the mode through the FSM accessor.
