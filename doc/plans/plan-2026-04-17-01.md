# Plan 01 — Real CLAP embeddings + PQ codebook training

## Goal

Ship the Python side of the embedding pipeline so `rfg --similar` returns
perceptually useful results on real audio. Replaces the synthetic random
vectors produced by `scripts/inject_synthetic_embeddings.py`.

## Context

Rust infrastructure is already in place:

- `samples.embedding BLOB` column (`src/engine/sqlite.rs`).
- `metadata` table for codebook + codebook-version storage.
- `pq::ProductQuantizer::from_bytes` (`src/engine/pq.rs:43`) accepts a
  `M * K * DSUB * 4 = 524288`-byte little-endian f32 blob with layout
  `centroids[(m * K + k) * DSUB .. +DSUB]` (m outer, k middle, d inner).
- Brute-force + ADC search paths light up automatically once
  `samples.embedding` is populated and `metadata.pq_codebook` exists
  (`src/engine/api.rs:88`, `src/engine/mod.rs:756`).

**No Rust changes in this sprint.**

  training pipeline sketch.
- `doc/designs/embedding-agentic.md` — agent-facing shape; Plan 1 is the

### 2. `scripts/embed_preprocess.py`

Shared preprocessing: load audio → resample 48kHz → mono → trim silence
(-60dBFS) → window to 1–3s → peak-normalize -1dBFS → return `np.float32`.

Public API:

```python
def preprocess(path: Path | str, target_sr: int = 48000,
               window_seconds: float = 2.0) -> np.ndarray:
    """Return mono float32 audio ready for CLAP inference."""
```

Errors return None; caller skips the row.

### 3. `scripts/embed_encode.py`

```
usage: embed_encode.py [--db PATH] [--batch-size N] [--limit N] [--dry-run]
```

Algorithm:

```python
rows = conn.execute("""
    SELECT id, path FROM samples
    WHERE embedding IS NULL
      AND (category IS NULL OR category NOT LIKE 'LOOP%')
    LIMIT ?
""", (limit,)).fetchall()

for batch in chunks(rows, batch_size):
    audios = [preprocess(p) for (_, p) in batch]
    # Drop rows that failed preprocess; shift indices.
    valid = [(rid, a) for (rid, _), a in zip(batch, audios) if a is not None]
    if not valid:
        continue
    audio_batch = np.stack([a for _, a in valid])
    embeddings = clap.get_audio_embedding_from_data(audio_batch, use_tensor=False)
    # embeddings: (batch, 512) float32
    for (rid, _), emb in zip(valid, embeddings):
        blob = emb.astype('<f4').tobytes()  # 2048 bytes
        conn.execute("UPDATE samples SET embedding = ? WHERE id = ?",
                     (blob, rid))
    conn.commit()
```

Model path taken from env `LAION_CLAP_CHECKPOINT` or a flag; default
`~/Library/Application Support/riffgrep/models/music_audioset_epoch_15_esc_90.14.pt`.

Idempotent because of `WHERE embedding IS NULL` gate.

### 4. `scripts/embed_train.py`

```
usage: embed_train.py [--db PATH] [--n-train N] [--seed N]
```

Algorithm:

```python
rows = conn.execute("""
    SELECT embedding FROM samples
    WHERE embedding IS NOT NULL
      AND category NOT LIKE 'LOOP%'
    ORDER BY random_seeded(?) LIMIT ?
""", (seed, n_train)).fetchall()

vecs = np.stack([np.frombuffer(b, dtype='<f4') for (b,) in rows])
# shape (N, 512)

pq = faiss.ProductQuantizer(512, 128, 8)   # M=128, nbits=8 → K=256
pq.train(vecs)

# Extract centroids in (M, K, dsub) = (128, 256, 4) row-major order.
centroids = faiss.vector_to_array(pq.centroids).astype('<f4', copy=False)
assert centroids.nbytes == 524288, centroids.nbytes
blob = centroids.tobytes()

# Bump version atomically with codebook replace.
with conn:
    cur = conn.execute("SELECT value FROM metadata WHERE key = 'pq_codebook_version'")
    row = cur.fetchone()
    version = (int.from_bytes(row[0], 'little') + 1) if row else 1
    conn.execute("INSERT OR REPLACE INTO metadata (key, value) VALUES ('pq_codebook', ?)", (blob,))
    conn.execute("INSERT OR REPLACE INTO metadata (key, value) VALUES ('pq_codebook_version', ?)",
                 (version.to_bytes(8, 'little'),))
```

### 5. Tests

`scripts/tests/conftest.py` — fixtures for an in-memory SQLite with the
minimum schema riffgrep expects.

`scripts/tests/test_codebook_rust_compat.py` — builds a tiny codebook
with hand-crafted distinguishable centroids, serializes via the same
byte path as `embed_train.train()`, and verifies the bytes decode back
with a Python reimplementation of Rust's `pq::ProductQuantizer::from_bytes`
offset formula (`centroids[(m * K + k) * DSUB .. +DSUB]`). This proves
the Python-side layout matches what Rust's decoder expects. The Rust
side of the round-trip (`from_bytes → to_bytes` symmetry) is covered by
`pq::tests::codebook_roundtrip` in `src/engine/pq.rs`; together these
validate both halves of the format contract. (P1.1.)

`scripts/tests/test_encode_idempotent.py` — runs `embed_encode.main`
twice on a fixture DB where the first run fully populates it; asserts
the second run issues zero UPDATE statements. Mocks `laion_clap` to
return deterministic vectors (no model dep). (P1.2.)

`scripts/tests/test_ranking_sanity.py` — **marked
`@pytest.mark.requires_clap_model`**; skipped unless env
`LAION_CLAP_CHECKPOINT` is set and a fixture dir exists. Builds a hand-
labeled 50-file corpus, runs `embed_encode` with the real CLAP encoder,
then scores pairwise L2 rankings in Python over the stored
full-precision embeddings. No PQ training, no `rfg --similar` CLI
invocation — those belong to Plan 3. Assertion: ≥80% of queries have
≥8/10 top neighbors matching the query's category label. Calibrated
threshold locked after first green run. (P1.3.)

`scripts/tests/test_embedding_norm.py` — uses same CLAP-backed fixture
OR a mock, asserts every stored embedding has `||v||_2 ∈ [0.9, 1.1]`.
(P1.4.)

`scripts/tests/test_loop_skipped.py` — fixture DB with 5 rows,
category='LOOP/Breaks' on 2 of them. Mocked encoder. Runs
`embed_encode.main`. Asserts `SELECT COUNT(*) WHERE category LIKE
'LOOP%' AND embedding IS NOT NULL = 0`. (P1.5.)
