## Tasks

### T1 — `stop()` unconditionally takes the sink

**Problem:** `PlaybackEngine::stop()` (`src/engine/playback.rs:764`)
only calls `sink.stop()` when the FSM emits
`Some(MixerCommand::Stop)`. The FSM returns `None` when transport is
already `Stopped` — for example after `state()`'s drain-grace path
dispatched `Input::ProgramEnded` on natural program end. In that
window the sink is still allocated; gating the cleanup on the FSM
output leaks it.

**Solution:** Dispatch `Input::Stop` for FSM bookkeeping (transport +
`pending_*` clears), then unconditionally take the sink from the
`Option<Sink>` and call `sink.stop()` if it was `Some`.

### T2 — `restart_program()` no longer hard-zeros `sample_offset`

**Problem:** `PlaybackEngine::restart_program()` writes
`sample_offset = 0` immediately, but in reverse mode the mixer's
restart origin is `first.end - 1`, not `0`. This makes the UI
cursor snap to 0 for 1–2 ticks until the next
`update_sample_offset()` reads the mixer-updated `control.frame` and
syncs.

**Solution:** Drop the `sample_offset = 0` line. The mixer writes
the correct origin to `control.frame` when it consumes
`pending_restart`, and the next `update_sample_offset()` tick picks
it up. `paused_elapsed` and `play_start` stay — they're wall-clock
bookkeeping, unrelated to frame position.

### T3 — `doc/plans/plan-2026-04-22-02.md` drift corrections

Three in-repo doc inconsistencies flagged in round 2:

- **Context paragraph** (Plan 07 plan, line 14–17 pre-fix): reads as
  if `#[allow(dead_code)]` is *currently* on `Input` / `MixerCommand`
  / `PlaybackFsm`, but Plan 07 itself removed/narrowed that coverage.
  Reframe as "before this sprint ... this sprint removes and
  narrows..."
- **T1's `PlaySegment` snippet**: shows `pub reps: u32` but the
  real struct is module-private with `reps: u8` (255 = infinite loop).
  Correct the snippet.
- **T3's solution text**: claims the XOR is stored on
  `SourceControl::reversed`, but the implementation keeps the atomic
  as the bare global flag and computes the XOR per frame via
  `SegmentSource::effective_reversed()`. Rewrite to match reality.
